# CK工程 ISO/IEC 27001 啟動評估 + 認證路線圖
## ISMS Readiness & Certification Roadmap (2026-06-17)

> 對標：ISO/IEC 27001:2022（含 Annex A 93 控制）。
> 結論先行：**CK 嘅「技術控制」對一個 solo SaaS 嚟講異常強（~60-70% 已落地）,但「管理體系（ISMS）」層接近零。** ISO 27001 認證嘅係**管理體系,唔係 code** —— 主要差距係文件 + 管治流程,唔係技術。

---

## 一、點解值得做
1. **DWSS §2.1.3 點名** ISO/IEC 27001 —— 對齊政府數碼監督標準。
2. 由「**自述**」變「**第三方證明**」—— 正中「需要更多認證及證明」嘅核心訴求。
3. 投標 / 企業客戶 / 政府基金 嘅信任門檻。

**但要誠實**：CK **而家唔可以**聲稱「ISO 27001 認證／合規」。只有認可認證機構（UKAS/HKAS-accredited）經 Stage 1+2 審核先可發證。CK **今日可以**聲稱嘅見下「可即聲稱」一節。

## 四、認證路線圖（solo 創辦人現實版）

| 階段 | 內容 | 時間 | 成本 |
|---|---|---|---|
| **Phase 0 基礎** | 堵 3 個技術 gap（備份/真MFA/帳戶MFA）+ 簽 DPA + 資產登記 | 2-4 週 | ~US$25/月(Supabase Pro),其餘免費 |
| **Phase 1 ISMS 範圍+政策** | 定範圍(CK SaaS + Supabase)、context、政策(用 ISO27k 範本)、委自己做 ISMS owner（滿足 clause 4-5）| 3-6 週 | 免費(範本) |
| **Phase 2 風險評估 + SoA** | 對資產做風險評估、定處置、出 SoA(逐一交代 93 控制適用/排除 + 現狀,你 code 俾到約 10 個技術控制嘅「已實施」證據)（clause 6+8）| 2-4 週 | 免費 |
| **Phase 3 營運生證據** | 跑控制、記威脅情報、部署 verify_integrity cron、access review log（審核員要約 3 個月運行證據）（clause 7-9）| 8-12 週 | 低 |
| **Phase 4 內審 + 管理覆檢** | 對 SoA 內審(可自做或請平價顧問)、管理覆檢、記不符合 + 糾正（clause 9,認證前必須）| 1-2 週 | 低–中 |
| **Phase 5 認證審核** | 認可機構 Stage 1(文件) → Stage 2(實施) | — | **約 US$5k-12k**(Stage1+2)+ 年度監督審核 + 第3年重認證 |

**總計**：約 **4-7 個月**、**low-thousands USD**（自己做文件 + 用範本）。備份/真MFA/帳戶MFA 係真正提升安全（非純 paperwork）嘅優先。
**替代**：可先做平價 gap assessment 或較輕量認證（如 Cyber Essentials 式）再決定是否全 ISO。

## 六、可即聲稱 vs 不可聲稱

**✅ 今日可誠實聲稱：**
> 「建基於 ISO 27001 / SOC 2 認證基建（Supabase）；資料傳輸 TLS 1.2+、靜態 AES-256 加密；資料庫層 row-level security 強制角色存取；防篡改、append-only 審計軌跡；按項目資料隔離；通過 Apple 審核嘅帳戶刪除。」

**❌ 不可聲稱**（直至認可機構發證）：「ISO 27001 認證」「ISO 27001 compliant」「DWSS 認證」。
