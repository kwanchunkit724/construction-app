## 2. 評審五項答案（照貼，按實況微調）

### 創新與創意（25%）
CK工程（關春傑工程管理）係**第一個真正為香港地盤判頭而設、全廣東話**嘅工地管理 app。核心創新唔係單一功能，而係一個**「打得贏官司嘅審計記錄」**理念 —— 而呢個理念**有真正嘅技術後盾**：
- 每一個工地指令、變更單、工作許可證、圖則、進度更新、問題，連同「邊個、幾時」一齊**永久記錄**。
- 全系統有一條**防篡改 hash 鏈審計帳本**：改或者刪任何一筆舊記錄，條鏈即刻斷，一驗即知 —— **連用後台（資料庫）改都唔例外**。業界冇對手做到呢層。
- 高風險操作（批核 / 簽 PTW / 改角色 / 刪帳號）要**二步驗證**：就算密碼俾人偷咗，冇你部機都做唔到。

深度本地化係護城河：PTW 用香港術語（**動火證**、火警看守 fire-watch、高空、吊運）、變更單以**港幣**計、八種地盤角色按**每個工地**獨立權限。香港地盤一直靠 WhatsApp + 紙 + 記憶，一嘈交冇證據；CK 將呢啲全部收喺一個**改唔到、查得返**嘅系統。

### 功能（20%）
一個系統涵蓋判頭日常全流程：
- **進度表**：大／中／細項樹狀，**五種模式**配唔同工程 —— 百分比、樓層、清單（小型工程）、量度（渠務計米數）、單位狀態（大樓維修，逐單位狀態機 + 距法定限期倒數）。
- **工地指令 → 變更指令**：版本化、**伺服器計港幣金額**（客戶端改唔到）、可設審批鏈、批核後鎖死。
- **工作許可證（PTW）**：動火／高空／吊運，電子簽核鏈 + **QR 掃描核實（每次掃描留審計）** + 火警看守計時。
- **問題追蹤**：判頭→總承建商→PM 自動升級鏈，每宗有完整處理紀錄。
- **文件系統**：物料送審／施工方案／圖則／檢驗記錄，送審→批准／拒絕、逐項連住進度表、送審死線提醒、一鍵重新送審。
- **每日日誌**（出勤人數/機械/上下晝天氣/天文台警告）、物料、行事曆、聯絡人、平安咭、離線查閱。

### 市場潛力／表現（20%）
香港有**數以千計**註冊判頭／專門承建商（SME），全部係目標客。已**上 iOS App Store**（持續更新至 v1.4），Android **正式版存取權申請中**（已過 14 日封測 gate）。增長槓桿：
- **CITF 基金**：客戶買 app 可獲政府補貼 **70–90%**（中小企比率自 2026/5 再 +10%）—— 等於政府幫你打折賣俾客。
- **政策強制**：DEVB 規定 **>$30M 工程由 2026/4 起要用數碼監工系統（DWSS）** —— CK 等於判頭級數碼監工。
- 之後拓展**大灣區**。
產品已部署於 9 個試點地盤，由真實判頭日常驗證。

### 效益與影響（20%）
- **慳錢／避風險**：地盤付款爭議，邊個記錄好邊個贏。CK 將呢個變成**可量化效益**（避免嘅索償成本）—— 而且記錄係**防篡改**嘅，對方賴唔到、改唔到。
- **ISO 9001 副產品**：判頭日日用 CK，自動產生 ISO 9001 級「記錄控制、可追溯、改善流程、不可否認（non-repudiation）」證據 —— 接政府工程（DEVB 認可承建商名單要 ISO）直接加分。新加嘅 hash 鏈帳本 + 二步驗證簽核，正正係審計師想見嘅「tamper-evidence + 身份綁定」。
- **安全**：PTW 電子簽核 + QR 核實，動火等高危工序有合規記錄；簽核要二步驗證，責任綁死真人。
- **切合政府方向**：DWSS、智慧工地、ISO —— 全部對得上。

### 質素（15%）
- 每張資料表都有 **RLS 權限控制**，經**兩輪正式 security hardening**，再加 2026-06 一次**全面保安升級**：
  - **防篡改 hash 鏈審計帳本** + `verify_integrity()` 一鍵驗證 + 可導出加密證明（第三方離線核實）。
  - **二步驗證（TOTP MFA）step-up**：8 個高風險伺服器操作就算用 raw API 都繞唔到（伺服器強制，唔係前端扮嘢）。
  - 修補咗一個**角色提權漏洞**（被入侵嘅 PM 都改唔到成員角色）。
- 符合 **Apple 帳號刪除**規定（保留審計記錄）。
- 進度歷史**不可改**（append-only）+ 期間快照。
- 金額由**伺服器**計，客戶端改唔到。
- 完整測試：每個保安機制都用真實角色喺正式環境**執行驗證**（唔係淨係睇 source）。

## 3. 保安與合規 一頁紙（One-Pager，可獨立交 / 放 appendix）

> **「審計記錄打得贏官司」嘅技術後盾。**

**A. 防篡改（Tamper-evidence）**
每一個關鍵改動（簽核、進度、文件、成員角色…）即時封入一條 **hash 鏈審計帳本**：每筆記錄存住前一筆嘅 sha256 指紋，環環相扣。改或刪任何一筆舊記錄 → 條鏈即刻斷。`verify_integrity()` 一 click 行勻成條鏈，綠色「完整」或紅色「喺第 N 筆斷咗」。**連用 Supabase 後台 / service key 改都會被偵測到**（trigger 對所有寫入生效）。可導出加密證明俾第三方離線核實。誠實定位：**tamper-EVIDENT（查得到）**，唔係吹「改唔到」。

**B. 身份認證（二步驗證 step-up）**
高風險操作（批 SI/VO、簽 PTW、改角色、改審批鏈、文件審批、刪帳號）要**二步驗證**：TOTP 驗證碼 → session 升 AAL2 → 5 分鐘授權窗。伺服器端每個 RPC 開頭強制檢查，**raw API 都繞唔到**。賊仔有 phone+password 都批唔到、簽唔到、刪唔到。安全 rollout：用 flag 控制，client 全平台上齊先開。

**C. 權限與基礎**
- 每張表 RLS，兩輪 hardening + 2026-06 升級。
- 角色提權漏洞已修（guard trigger）。
- 金額伺服器計、進度歷史 append-only + 快照。
- PTW QR 每次掃描留審計 + 簽名伺服器寫入（偽造唔到）。
- Apple 帳號刪除合規（FK cascade）。

**技術棧**：React + TypeScript + Capacitor（iOS/Android）+ Supabase（Postgres + RLS + Auth MFA）。單一後端，易加 AI 層。
