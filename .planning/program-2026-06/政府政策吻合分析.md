## 一、政府推緊乜（你要對標嘅四件嘢）

### 1. Construction 2.0 —「時、勢、機」三大支柱
- **創新 (Innovation)** — 數碼化、MiC、BIM、創新物料／技術、智慧工地。
- **專業化 (Professionalisation)** — 標準化流程、質量保證、規管、安全。
- **年青化 (Revitalisation)** — 吸引年青人、改善工地形象、易用工具。
- 政策目標明文：**提升生產力、質量、工地安全、減少環境影響**。

### 2. DWSS 數碼工程監督系統（TC(W) 2/2023）← 政府最具體嘅「數碼標準」
- 適用：> $3,000萬基本工程合約（2023-04-01 起），門檻由 $3億降到 $3,000萬 = **政府明確想數碼化落到中型工程**。
- 本質：**一個中央化、workflow 驅動嘅工地監督平台**，網頁 + 原生 App，數據自動同步，連住 iCWP。
- **6 個強制模組**：① RISC 檢驗/測量申請表 ② 工地日誌/記錄簿 ③ 工地安全檢查記錄 ④ 清潔檢查清單 ⑤ 勞工申報 (G.F.527) ⑥ 合約管理 (NEC：補償事件/PM指令/預警/付款/工序)。
- **+ 智慧工地應用模組**：集中 IoT／AI鏡頭／感應器／SSSS（智慧工地安全系統）數據 → dashboard + 實時警報。
- **保安規格（= 你想要嘅「認證及證明」核心）**：登入雙因素認證、操作前重認證、增改刪審計軌跡(留 6 個月)、256-bit SSL、傳輸+靜態加密、ISO/IEC 27001、Tier-3 機房、獨立審計報告、MDM、提交後鎖定、唯一識別碼、時間戳+操作人、API/Web Services 24h 內供 iCWP 抽取、BIM 相容、英文 DB 儲存、離線暫存+自動同步、原生 iOS/Android app、備份+災難復原。

### 3. iCWP 綜合基本工程平台 + 數據標準化
- 政府將各 DWSS + 數碼系統嘅數據標準化、自動上傳 iCWP，做跨項目監控 + 分析。
- 含義：**政府要嘅唔淨係「電子化」,係「結構化、可抽取、可互通」嘅數據。**

### 4. 工務工程應用研發（$3,000萬基金,2022-23 預算）
- 鼓勵工務工程 + 業界採用**新物料、創新建造技術**,目標：減成本、慳時、提生產力、廣泛適用。
- 2021 設「工務工程應用研究專責小組」：試點 → 訂標準 → 推廣。
- 含義：政府用基金 de-risk 創新採用；CK 屬「提升生產力嘅創新工具」可入呢個敘事（配合你已申請嘅 CITF／HKICTA／Smart Lab）。

## 二、CK 已經吻合咩（強項 — 可即用作賣點同「證明」）

DWSS 規格其實係一張 checklist,CK 已經命中相當多 —— 對一個**私營判頭 App** 嚟講係好強。

### 6 大強制模組對照
| DWSS 模組 | 政府要求 | CK 現狀 | 評 |
|---|---|---|---|
| ② 工地日誌/記錄簿 (G.F.536) | 進度/人手/機械/天氣/事件,審閱簽認,鎖定 | **每日施工日誌**：出勤、機械、上/下晝天氣、天文台警告、進度項目、備註;當日鎖定 | 🟢 強 |
| ⑥ 合約管理 (PM指令/變更) | 指令、變更、金額、審批 | **SI 工地指令 + VO 變更指令**（伺服器計港幣）+ 多級審批鏈 + 批後鎖定 | 🟢 強（NEC 細項除外）|
| ③ 工地安全記錄 | 安全檢查、不合格記錄、相、簽核 | **PTW 工作許可證**（動火/高空/吊運/密閉/掘地/電力/棚架/升降機,checklist+工人+相+安全主任簽核+QR核實+火警看守）+ 問題追蹤 | 🟡 部分強 |
| ① RISC 檢驗申請 | 申請→檢驗→批,連 ITP,相,歷史 | **圖則/文件送審審批** + 問題追蹤 + 進度檢核（近似但非正式 RISC/ITP）| 🟡 部分 |
| ⑤ 勞工申報 | 人手部署 + 工資率月報 | 日誌有出勤/人手,**未有 G.F.527 月報 + 工資率** | 🟡 部分 |
| ④ 清潔檢查 | 每日/每週清潔檢查清單 | 無 | 🔴 缺 |
| + 智慧工地模組 | IoT/AI/感應器/SSSS 整合 | **AI 站長**（廣東話查工地、實時天氣);未整合 IoT/感應器/AI鏡頭 | 🔴 缺（AI 起步)|

### 保安／證明規格對照（你最關心嗰part）
| DWSS 要求 | CK 現狀 | 評 |
|---|---|---|
| 操作前重認證（提交/批核）| **簽名證本人**（密碼重認證 + 簽名證書）| 🟢 強（直接命中）|
| 增改刪審計軌跡（留 6 個月）| **防篡改 hash 鏈審計帳本 + 一鍵 verify + 匯出證明** | 🟢🟢 超標 |
| 提交後鎖定,改靠 reject/resubmit | SI/VO/PTW/文件 批後鎖定 | 🟢 ✓ |
| 提交/批核 時間戳 + 操作人 | 審批鏈逐步記錄 actor + 時間 | 🟢 ✓ |
| 256-bit SSL、傳輸+靜態加密 | Supabase TLS + at-rest 加密 | 🟢 ✓（底層）|
| 原生 iOS/Android App | Capacitor,iOS App Store live + Android | 🟢 ✓ |
| 匯出 PDF/XLS | Excel + PDF 報告 | 🟢 ✓ |
| 角色/用戶權限管理 | 8 角色 RBAC + 每項目成員 + RLS 後端強制 | 🟢 ✓ |
| 推送通知（逾期/指派）| OneSignal 審批/簽核推送 | 🟢 ✓ |
| 模組可開關（按項目派模組）| **每項目模組開關**（admin） | 🟢 ✓ |

> 重點：DWSS §2.5（審計軌跡）+ §3.1.7（操作前重認證）+ §3.1.11（鎖定）+ §3.1.12（時間戳）—— 呢幾條正正係你近排做嘅 **審計帳本 + 簽名證本人 + 二步驗證**,CK 喺呢度甚至**超出** DWSS 嘅最低要求（hash 鏈防篡改, DWSS 只要求 log）。呢個係最強嘅「認證及證明」賣點。

## 四、「認證及證明」改進路線（你嘅核心訴求）

分三層,由**即賺分**到**長線**：

### A. CK 已有底子 → 只需「包裝成證明」（即做,最高 ROI）
1. **合規證明書 (Compliance Proof Sheet)** — 列明 CK 對齊 DWSS 邊幾條（重認證/審計鏈/鎖定/時間戳/加密/RBAC),附審計帳本 verify 報告 + 簽名證書樣本。一頁 PDF,投標/客戶/獎項即用。
2. **簽名證書 + 審計鏈 verify** 已 live → 做個「匯出合規包」按鈕（每張 PTW/SI/VO 出單一 PDF + hash 證明),正正對標 DWSS §5.4 交付規格。
3. 把「防篡改帳本」「簽名證本人」寫成**對標 DWSS §2.5/§3.1 嘅一句話**,放入 /sell 同 pitch deck。

### B. 補規格差距 → 真·提升對齊（中期,揀高價值先）
- **優先**：① 登入 2FA（生物認證,易,合規分高） ② 相片 GPS+時間 metadata（證據力,易） ③ 統一文件唯一識別碼 ④ 英文 i18n（國際/合規門檻）。
- **次**：離線寫入同步、對外 API（iCWP-ready 賣點）、清潔檢查 + 週/月安全巡查模組（補 DWSS 模組）。

### C. 對外第三方認證／認可（真·硬證明,長線）
- **ISO/IEC 27001**（資訊安全)— DWSS 點名要求;CK 有審計鏈+加密+RBAC 底子,係最對口嘅認證目標。
- **CITF 預批名單**(進行中)— 政府基金認可 = 最直接「政府背書」。
- **HKICTA / Smart Lab**(已申請)— 創新獎項/政府採購曝光。
- 長線：OGCIO 認可、考慮對接 iCWP 數據標準做「DWSS-compatible」標籤。

## 五、定位建議（最重要嘅一句）

**CK ≠ 政府 DWSS。** DWSS 係承建商按每張 >$3,000萬合約自費採購、HKSARG 擁有、合約完結要交還政府嘅系統。CK 唔應該扮 DWSS。

**CK 應定位為：Construction 2.0 對齊、DWSS-inspired 嘅私營/中小判頭數碼工地管理平台** —— 服務政府 mandate **覆蓋唔到嘅市場**（sub-$3,000萬工程、私營項目、判頭/判頭工人層）。

策略：
- 用 **DWSS Annex A 做 CK 嘅「質量框架」**（免費現成嘅功能 roadmap + 規格),逐步對齊 = 自動提升專業度。
- 把已有嘅 **審計鏈 + 簽名證本人 + 鎖定 + 時間戳** 包裝成「達到/超出政府 DWSS 數碼監督標準嘅證明」—— 呢個係中小判頭市場冇人有嘅信任武器。
- 攞 **ISO 27001 + CITF 認可** = 由「自述」變「第三方證明」,正中你「需要更多認證及證明」嘅訴求。

**一句總結：CK 喺 Construction 2.0 嘅「創新+專業化」方向已經行得好前(數碼化、AI、防篡改證據鏈);差嘅唔係功能,係「對外證明」—— 補登入2FA+相片元數據+英文,再攞 ISO 27001 + CITF 認可,就由「好用嘅 App」升級成「可證明合規嘅平台」。**
